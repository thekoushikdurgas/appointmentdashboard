# Deploy log analysis (resolved in repo)

## Real failure
- Docker: "failed to bind host port 0.0.0.0:3000/tcp: address already in use"
- Cause: `docker compose down` only stops Compose containers. A host process was still listening on :3000.
- Evidence: `ss` / `lsof` showed `next-server` (Next.js) on PID holding *:3000 — e.g. an old `next start` / PM2 / systemd unit outside Docker.

## Fix applied
- `.github/workflows/deploy.yml`: after `compose down`, free TCP 3000 via `sudo fuser -k 3000/tcp` (or `lsof` + `kill` fallback), then `compose up`.

## Not errors (informational)
- Lines prefixed `err:` from docker compose (Stopping/Removing/Creating) are normal stderr from the CLI.
- Next.js build: "Experiments (use with caution)" — expected when `experimental` is set in `next.config.ts`.
- npm "New major version available" — optional global npm upgrade; does not fail the build.

## EC2 hygiene (recommended once)
- Disable or remove any non-Docker service that runs Next on port 3000 (systemd, PM2, screen) so only Docker publishes :3000.
- Optional: `sudo apt-get install -y psmisc` if `fuser` is missing (rare on Ubuntu server images).
