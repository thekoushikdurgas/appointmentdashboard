# Deploy log — issues found and fixes (contact360.io/app)

## 1. Error: npm EACCES on `node_modules/.bin` (fatal)
- **Log:** `npm error code EACCES` / `permission denied, rmdir '.../node_modules/.bin'`
- **Cause:** `node_modules` (or the repo tree) was created or modified by **root** (e.g. earlier **Docker** bind-mount builds, or `sudo npm`). The SSH user (e.g. `ubuntu`) cannot delete/replace those paths during `npm ci`.
- **Fix (code):** `deploy/ec2-github-deploy.sh` now runs **`sudo chown -R "$(id -un):$(id -gn)" "$APP_DIR"`** after freeing port 3000 and **before** `npm ci`.
- **Fix (server):** If `sudo chown` fails in CI, grant the deploy user **passwordless sudo** for maintenance, or run once: `sudo chown -R ubuntu:ubuntu /home/ubuntu/appointmentdashboard` (adjust user/path).

## 2. Warning: garbled line `15477[STEP]` in logs
- **Cause:** `fuser -k` writes **killed PIDs to stdout**, which merged with the next `[STEP]` line.
- **Fix (code):** Redirect **`fuser`** output to `/dev/null` in `ec2-github-deploy.sh`.

## 3. Not errors
- **`err:`** lines from `git fetch` are normal stderr from Git.
- **`Build container for appleboy/ssh-action`** is the action’s Docker image, not your app stack.
